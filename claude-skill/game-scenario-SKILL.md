# 게임 시나리오 & 세계관 창작 스킬

## 개요

장르에 맞는 게임 시나리오, 세계관, 캐릭터, 퀘스트를 체계적으로 창작한다.
아이디어 한 줄만 줘도 완성도 있는 게임 설정 문서를 만들어준다.

---

## 창작 프로세스

### Step 1. 요청 파악

사용자 메시지에서 다음 정보를 추출한다. 명시되지 않은 항목은 창의적으로 결정한다.

| 항목 | 추출 방법 |
|------|-----------|
| 장르 | 명시된 경우 그대로, 없으면 요청 분위기에서 추론 |
| 분위기 | 키워드(어둡다, 밝다, 잔인하다, 따뜻하다 등)로 파악 |
| 요청 범위 | 세계관 전체 / 캐릭터만 / 퀘스트만 / 스토리 전체 등 |
| 참고 작품 | 언급된 게임·영화·소설이 있으면 레퍼런스로 활용 |

### Step 2. 장르별 핵심 가이드 적용

요청 장르에 따라 아래의 핵심 가치를 우선순위에 두고 창작한다.

- **RPG/어드벤처**: '성장'과 '수집'. 캐릭터의 서사와 세계관의 깊이에 집중한다.
- **전략/시뮬레이션**: '자원'과 '상성'. 세력 간의 균형과 시스템적 규칙에 집중한다.
- **액션/FPS**: '긴박감'과 '타격감'. 짧고 강렬한 목표와 직관적인 빌런에 집중한다.
- **퍼즐/인디**: '독창성'과 '메시지'. 독특한 기믹이나 감성적인 스토리텔링에 집중한다.

### Step 3. 산출물 생성

요청 범위에 따라 아래 모듈 중 필요한 것만 생성한다.

---

## 산출물 모듈

### 🌍 세계관 설정
```
## 세계관: [이름]

### 핵심 설정
- 배경: [시대·공간·문명 수준]
- 세계의 규칙: [마법 체계 / 기술 수준 / 물리 법칙 변형 등]
- 핵심 갈등: [세계를 움직이는 근본적인 대립 구도]

### 세력 구도
| 세력명 | 목표 | 특징 |
|--------|------|------|
| ...    | ...  | ...  |

### 지역
- [지역명]: [특징 1~2줄]
```

### 👤 캐릭터 설정
```
## 캐릭터: [이름]

- 역할: 주인공 / 빌런 / NPC / 파티원
- 한 줄 소개: [이 캐릭터를 한 문장으로]
- 배경: [출신, 과거, 동기]
- 성격: [3가지 키워드]
- 시스템 연계(기믹): [예: 겁이 많은 성격 → 적과 멀어질수록 공격력 상승 / 불을 다루는 혈통 → 화염 면역 및 주변 밝히기]
- 능력 / 특기: [게임 내 역할과 연결]
- 비밀 또는 반전: [스토리에서 드러나는 숨겨진 면]
```

### 📜 메인 스토리라인
```
## 스토리: [제목]

### 3막 구조
**1막 — 발단**
[플레이어가 사건에 휘말리는 계기. 1~3문장]

**2막 — 전개 & 위기**
[핵심 갈등이 고조되는 과정. 주요 반전 포함. 3~5문장]

**3막 — 결말**
[해결 방식과 엔딩 분기. 가능하면 2가지 이상 제시]
```

### ⚔️ 퀘스트 설계
```
## 퀘스트: [이름]

- 유형: 메인 / 서브 / 숨겨진
- 트리거: [어떤 조건에서 시작되는가]
- 목표: [플레이어가 해야 할 것]
- 핵심 선택지: [도덕적 딜레마 또는 전략적 분기]
- 보상: [아이템 / 정보 / 관계 변화]
- 결과 분기: [선택에 따른 세계 변화]
```

### 👹 보스 / 빌런 설계
```
## 빌런: [이름]

- 동기: [왜 악당이 됐는가 — 납득 가능한 이유]
- 철학: [빌런이 옳다고 믿는 것]
- 주인공과의 관계: [대비 또는 연결고리]
- 전투 콘셉트: [어떤 방식으로 싸우는가]
- 페이즈별 연출: [예: 체력이 50% 이하가 되면 분노하며 지형을 파괴(환경 메카닉 활용)]
- 약점 또는 반전: [클라이맥스에서 드러나는 것]
```

## 공통 창작 원칙

좋은 게임 시나리오의 핵심 원칙. 장르 참조 파일이 없거나 복합 장르일 때 이 원칙으로 창작한다.

1. **납득 가능한 동기** — 빌런도, NPC도 자기만의 논리가 있어야 한다
2. **선택의 무게** — 플레이어 결정이 세계에 실제로 영향을 미쳐야 한다
3. **복선과 회수** — 초반 설정이 후반에 의미 있게 연결되어야 한다
4. **장르 톤 일관성** — 어두운 세계관이면 끝까지, 밝은 세계관이면 끝까지
5. **플레이어가 주인공** — 스토리가 NPC 중심으로 흘러가지 않도록

## 출력 규칙

- 요청 범위가 좁으면(예: "캐릭터 하나만") 해당 모듈만 생성한다
- 요청 범위가 넓으면(예: "게임 하나 통째로") 세계관 → 주요 캐릭터 → 메인 스토리 → 퀘스트 2~3개 순서로 생성한다
- 생성 후 마지막에 항상 다음 확장 옵션을 제시한다:
  ```
  ---
  ➕ 더 만들 수 있어요:
  - [ ] 서브 캐릭터 추가
  - [ ] 퀘스트 확장
  - [ ] 지역 상세 설정
  - [ ] 엔딩 분기 추가
  ```
- 요청 언어에 관계없이 항상 한국어로 출력한다
